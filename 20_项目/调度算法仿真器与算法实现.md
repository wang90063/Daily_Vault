# 调度算法仿真器与算法实现

## 背景
- 该项目承接 [[调度算法走读与说明文档]] 在 2026-04-03（首版交付）与 2026-04-08（评审讨论）后的后续主线：从“说明材料/走读”转向“可跑实现 + 可复现实验数据证明”。（关键反馈记录见 `00_收件箱/20260408.md` / [[20260408]]）
- 主目标是两件事：
  - 做一个仿真器：用仿真数值在关键场景下证明所提算法的优势
  - 落地最小实现：把提出的调度算法实现出来，形成可跑、可验证的最小版本
- 推荐沿用业务抽象：上行 AI agent 传输；优先关注“弱覆盖用户 + 数据无法在一个 TTI 传完导致窝包 + 用户数多导致无法连续调度引入固定时延”等更可能出现增益的场景假设。[[20260408]]
- 相关研究正文与机制分析：[[调度算法机制走读与问题分析]]
