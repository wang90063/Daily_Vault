# 深度复盘Skill

## Context (目标与背景)

为 OrbitOS 设计并实现一个"深度复盘"Skill，帮助用户记录每天做了什么、哪些做得好、哪些需要改进，覆盖工作和生活两个维度。

**目标:** 建立一套可持续运行的每日复盘工作流，明确输入、处理流程和输出格式。

**成功指标:**

- [x] 完成方法论调研（已补充 B 站 GRAI / KISS 材料，详见 [[深度复盘skill项目]] 与 [[深度复盘模型梳理]]）
- [x] 确定复盘工作流设计（输入项、默认扫描范围、处理逻辑、输出格式，详见 [[深度复盘Skill工作流设计]]）
- [ ] 实现 `/daily_review` skill 并可正常使用

**限制条件:**
- 依赖项: 先完成方法论调研，再设计工作流

---

## Actions (计划与步骤)

### 阶段1: 方法论调研

- [x] 收集 3-5 个复盘方法参考（Atlassian / KPT / ORID / AAR / B 站 GRAI-KISS 视频）
- [x] 整理各方法的核心框架和适用场景
- [x] 选定当前主模型：`GRAI + KISS`

### 阶段2: 工作流设计

- [x] 定义输入项（用户需要提供什么）
- [x] 设计处理流程（AI 如何帮助整理和分析）
- [x] 确定输出格式（复盘笔记的结构）

### 阶段3: Skill 实现

- [x] 编写 `/daily_review` skill prompt（MVP 已创建）
- [ ] 创建复盘笔记模板（已完成 [[Record_Template]]、[[LightReview_Template]]；后续补齐面向 skill 输出的正式复盘模板）
- [ ] 测试并迭代优化
