# PPT成果复盘SOP

## 背景

这个项目承接已归档的 [[3月30日成果梳理PPT]]，目标不是继续修改那次汇报内容，而是把本轮从启动、对齐、出稿、演练到交付的推进过程，沉淀成一套下次可直接复用的 SOP。核心问题是：在什么节点，我和 agent 要共同产出什么材料，怎样判断这一轮已经可以进入下一步。

本项目只保留“复盘并沉淀 SOP”这一条主线；关于本周口播训练的连续事项，仍留在 [[2026-03-30]] 与后续日记里推进，不纳入本项目范围。

**目标：** 形成一套可复用于后续汇报类 PPT 的推进 SOP，明确关键节点、产出物、协作方式与验收标准。

**成功指标：**
- [ ] 梳理出一版完整推进链路：需求澄清、素材准备、结构设计、页面文案、讲稿演练、交付前自审
- [ ] 为每个节点明确我与 agent 的共同产出物、输入依赖与验收标准
- [ ] 形成一份下次可直接照着执行的 SOP 主文档或清单

**约束条件：**
- 以 [[3月30日成果梳理PPT]] 的真实推进记录为基础，不凭空抽象
- 优先沉淀“节点和交付物”，不扩展到新的表达训练系统

## 行动

### 阶段1：回收本轮推进事实

- [x] 回看 [[3月30日成果梳理PPT]]、[[2026-03-30]]、[[20260330记录自己]]，抽出真实发生过的关键节点
- [x] 列出每个节点当时产出的材料，例如大纲、页面草稿、文案、口播稿、讲解要点、演练稿
- [x] 标出哪些节点推进顺畅，哪些节点反复返工

### 阶段2：抽象成可复用 SOP

- [x] 将本轮节点整理成稳定顺序：需求澄清 -> 结构设计 -> 内容填充 -> 统一收口 -> 演练反馈 -> 最终交付
- [x] 为每个节点定义“需要一起产出什么材料”“产出到什么程度才进入下一步”
- [x] 补充每个节点的常见风险、容易遗漏的检查项与建议节奏

### 阶段3：形成可执行模板

- [x] 输出一版可直接复用的 SOP 主文档
- [x] 如有必要，再拆成简版清单或下次 kickoff 可直接引用的模板入口
- [ ] 在项目进展中记录这版 SOP 首次可复用的判断依据
