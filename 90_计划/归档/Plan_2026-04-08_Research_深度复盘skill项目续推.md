# 研究计划: 深度复盘skill项目续推：实现蓝图与落地路径

## 研究目标
- 明确“深度复盘Skill”在 OrbitOS 中接下来的实现边界、输入/输出结构和默认扫描策略，确保它和已有的反思、进度更新、问答 skill 形成互补。
- 梳理现有工作流与知识库里可复用的概念，找出需要补充的 atomic notes 与例子，保证落地产出既示例驱动又具备可执行操作。

## 发现的上下文
- 相关领域：OrbitOS Skill 体系、复盘方法论（GRAI+KISS 混合、双代理+双维度复盘）以及 Skill 实现中常用的输入扫描/输出回写规范。
- 现有笔记：`20_项目/深度复盘Skill.md`、`30_研究/OrbitOS/深度复盘skill这个项目/深度复盘skill项目.md`、`30_研究/OrbitOS/深度复盘skill这个项目/深度复盘Skill工作流设计.md`、`40_知识库/复盘方法/复盘问题梯度.md` 等，已经涵盖模型、输入层、行动闭环与无责原则。
- 相关技能：`.agents/skills/reflect/SKILL.md`（复盘边界与防错）、`.agents/skills/update-progress/SKILL.md`（轻量进度同步）、`.agents/skills/ask/SKILL.md`（快速知识答疑）的本地 workflow 可作为参考，需对照触发词、输出格式与边界控制。
- 相关项目：`20_项目/深度复盘Skill.md`（active），其 Actions 描述当前进度待向 Skill 实现转化。

## 研究策略
- [ ] 重新梳理三份关键 Skill 的触发词、输入样式、输出结构与边界声明，找出和深度复盘 Skill 的异同，以设计清晰的触发信号与守门规则。
- [ ] 衔接现有研究与项目笔记，提炼需要补充的“可执行输入模版”“行动收敛层示例”“默认扫描范围/升维规则”，并检查知识库中已有的atomic概念是否足够。
- [ ] 设计主笔记与示例笔记的结构：一篇主笔记总结实现蓝图与落地路径，配套 1~2 个示例/模板（如 KISS 模式日复盘、GRAI 模式事件复盘）以供 Skill 调用。
- [ ] 明确回写规则：哪些输出要写入日记、项目 Progress，哪些要沉淀到知识库（新 concepts），还有 skill 执行流程中的“确认节点”。
- [ ] 验证现有模板/知识库是否支持“示例驱动可执行”的要求，必要时补充到 `40_知识库/复盘方法/` 下的新概念。

默认假设：若无额外澄清，则本轮按“面向 skill 落地、示例驱动与可执行优先”继续。
