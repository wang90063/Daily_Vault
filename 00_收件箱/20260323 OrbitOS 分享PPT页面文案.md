# OrbitOS 分享 PPT 页面文案

**页数建议:** 9 页  
**分享时长:** 12-15 分钟  
**定位:** 可以直接复制到 PPT / Keynote 的页面文案

## 第 1 页

**标题**
我为什么做 OrbitOS

**副标题**
不是为了做一个更复杂的笔记系统，而是为了解决两个很现实的问题

**正文**
- 需求 1：管理每日进展，支持日复盘、周复盘、月复盘
- 需求 2：不能只看零散进展，还要按项目追踪工作
- 目标：让 AI 真正接入我的日常工作流

**备注**
开场先讲需求，不要先讲工具。

## 第 2 页

**标题**
以前的问题到底是什么

**副标题**
不是记得不够多，而是记完以后没有形成流转

**正文**
- 只记日记，信息会越来越碎
- 只记项目，看不到每天怎么推进
- 想法、任务、研究、项目散落在不同地方
- 过几天以后，很难快速接上上下文

**备注**
这一页的目标是让听众先认同问题。

## 第 3 页

**标题**
第一个核心抓手：每日进展

**副标题**
用 `10_日记` 承接每天发生的事情，给复盘留下素材

**正文**
- 每日笔记不是心情记录，而是工作锚点
- 日复盘：看当天推进、卡点和明天要接的事
- 周复盘：看重复出现的问题和时间分配
- 月复盘：看哪些事情真的形成积累

**备注**
强调一句：没有每日素材，就很难做高质量复盘。

## 第 4 页

**标题**
第二个核心抓手：项目追踪

**副标题**
工作不是按时间推进的，本质上是按项目推进的

**正文**
- 用 `20_项目` 承接一件事的完整状态
- 项目至少包含：背景、动作、进展
- 项目笔记回答：目标是什么、现在在哪、下一步是什么
- 关键进展会回链到具体某一天的日记

**备注**
强调一句：只有进展，没有项目主线，最后还是会散。

## 第 5 页

**标题**
这两个视角怎么连起来

**副标题**
日记记录当天发生了什么，项目沉淀累计推进到了哪里

**正文**
- 新想法先进入 `00_收件箱`
- 开始一天时回到日记，或者用 `/start-my-day`
- 需要长期推进的内容进入 `20_项目`
- 重要推进从日记同步回项目 Progress

**备注**
这一页最好配一张流转图：收件箱 -> 日记 -> 项目 -> 复盘。

## 第 6 页

**标题**
AI 在这里起什么作用

**副标题**
AI 不是替我思考，而是替我维护系统

**正文**
- 帮我起草每日规划
- 帮我处理收件箱里的模糊想法
- 帮我把想法整理成项目
- 帮我把零散内容沉淀成结构化笔记

**备注**
不要讲成“AI 什么都能做”，而是“AI 让这套系统跑得动”。

## 第 7 页

**标题**
为什么同时用 Codex 和 Claude

**副标题**
两个 AI 入口，同一套工作流

**正文**
- 都在同一个 Vault 根目录工作
- 都看到同一套目录结构和 Markdown 文件
- `AGENTS.md` 负责高层规则
- `.agents/skills/` 负责工作流能力
- `Codex` 偏结构化执行，`Claude` 偏长对话探索

**备注**
重点讲“共用一套系统”，不要展开模型对比。

## 第 8 页

**标题**
我是怎么做 Git 同步的

**副标题**
内容同步一条线，系统开发同步一条线

**正文**
- 日常内容文件：直接在 `main` 提交，commit 前缀用 `content:`
- 系统文件改动：先建 `dev/功能名` 分支，再开发，再合并回 `main`
- 同步前先看 `git status`，只提交本次相关文件
- 不用 `git add .`，避免把无关内容一起带上去

**示例命令：日常内容同步**
```bash
git checkout main
git pull origin main
git status
git add 00_收件箱/ 10_日记/ 20_项目/
git commit -m "content: 更新分享演讲稿与PPT提纲"
git push origin main
```

**示例命令：系统开发同步**
```bash
git checkout main
git checkout -b dev/功能名
git status
git add AGENTS.md .agents/skills/ 99_系统/模板/
git commit -m "docs: 更新分享相关系统文档"
git push -u origin dev/功能名
```

**备注**
这页重点不是教 Git，而是说明这套系统怎么稳定同步，不把内容流和开发流混在一起。

这套系统最终帮我回答两个问题

**副标题**
今天我做了什么，以及我手上的关键项目推进到哪里了
