# 个人深度复盘示例

## 场景
- 时间范围：2026-03-18
- 复盘对象：个人工作日
- 背景：当天同时推进 [[深度复盘Skill]]、[[3月30日成果梳理PPT]] 和 [[分层分级技术图]]，下午出现注意力下降

## 原始输入示例
### 事实
- 上午完成了“深度复盘 Skill”方法论调研
- 下午没有推进 [[项目结构与原理深度探索]] 的 Task 3.1
- 晚上开始担心多个项目并行导致节奏失控

### 感受与精力
- 上午有明显掌控感
- 下午切到模糊任务时出现拖延
- 晚上更想收尾而不是继续思考

### 预期 vs 实际
- 预期：今天把两个中型任务都推进一轮
- 实际：只完成了调研，另一个任务停在“知道要做但没开工”

## Skill 提问链示例
### Objective
- 今天确定发生了哪些事件？
- 哪些任务完成了，哪些没有完成？
- 有没有证据材料可以引用？

### Reflective
- 今天最有能量的时刻是什么？
- 哪个环节开始分心？
- 我在回避什么类型的任务？

### Interpretive
- 为什么结构清晰的任务更容易推进？
- 下午失速的根因是疲劳、模糊，还是切换成本？
- 这种模式过去是否重复出现过？

### Decisional
- 哪个做法值得继续保持？
- 哪个环节需要限制或修正？
- 下次遇到相同触发条件时，我准备做什么？

## 输出示例
### 复盘概览
今天的高质量产出来自“任务边界清晰 + 反馈及时”，而失速主要发生在“目标模糊 + 缺少最小下一步”的时段。

### 工作维度
- Keep：先做结构清晰的任务时，推进效率明显更高
- Problem：下午切换到抽象任务时没有先拆最小动作，导致停滞
- Try：每天 14:00 前，为下午主任务先写一句“最小下一步”

### 生活维度
- 上午精力较高，适合做分析与写作
- 下午需要一个短暂重启动作，否则容易进入低质量分心

### 洞察与模式
1. 模糊任务不是不能做，而是不能直接做，需要先降维成一个具体动作。
2. 精力管理和任务设计要一起看，否则容易把节奏问题误判成意志力问题。
3. 如果深度复盘要长期使用，输出必须少而准，不能像问卷那样越写越重。

### 下一步实验
- 明天下午开始前先写出主任务的第一步动作
- 晚上复盘时只保留 1 条 Keep、1 条 Problem、1 条 Try
- 每周做一次深度版，每天只做轻量版
