# OrbitOS 目录结构与核心模块分析

## 概述
OrbitOS 是一个以用户为中心的知识管理与任务执行系统。其核心逻辑在于通过数字前缀的规范化目录结构，实现从“信息捕捉”到“项目执行”再到“知识沉淀”的闭环流转。

## 核心架构与模块功能

### 1. 捕捉与处理阶段
- **`00_收件箱`**: 系统的第一入口。用于快速捕捉所有未分类的想法、任务或信息碎片的临时存放区。
  - **流转逻辑**: 通过 `/kickoff` (转化为项目) 或 `/research` (转化为研究) 进行处理，处理后标记为 `processed`。

### 2. 执行与记录阶段
- **`10_日记`**: 每日的时间轴。记录当天的计划、进度和反思。
  - **关键工具**: 使用 `/start-my-day` 进行早间规划，承接昨日遗留并开启今日任务。
- **`20_项目`**: 当前活跃的攻坚点。采用 C.A.P. (Context, Actions, Progress) 布局。
  - **组织逻辑**: 扁平化管理，通过 frontmatter 中的 `area` 字段链接到知识库领域。
- **`90_计划`**: 执行过程中的详细蓝图。存放尚未归档的复杂执行计划。

### 3. 研究与沉淀阶段
- **`30_研究`**: 深入探索的结果。存放针对特定主题的深度调研笔记。
- **`40_知识库`**: 原子化的概念库。存放经过提炼的、可复用的单一知识点。
- **`50_资源`**: 外部信息的策展。例如通过 `/ai-newsletters` 或 `/ai-products` 自动生成的每日摘要。

### 4. 系统底层支撑
- **`99_系统`**: 系统的“代码层”。
  - **模板**: 确保所有笔记结构一致。
  - **提示词**: 定义 Agent 在不同场景下的专业化人格。
  - **归档**: 存放已完成的项目和过期的收件箱条目。

## Agent Skills 的角色
`.agents/skills` 是 OrbitOS 的“驱动引擎”。
- **交互逻辑**: 当用户输入指令（如 `/start-my-day`）时，Agent 激活对应的技能脚本。
- **职责**: 技能脚本定义了 Agent 如何读取目录中的信息、如何与用户确认、以及最终如何按照 `99_系统/模板` 的规范写入新数据。

## 最佳实践
1. **数字前缀驱动**: 严格遵守 00-99 的数字前缀，保持文件排序的逻辑性。
2. **Wikilinks 自由连接**: 不受文件夹层级限制，通过 `[[NoteName]]` 实现跨模块连接。
3. **C.A.P. 布局约束**: 确保每个项目都有清晰的上下文、可操作的步骤和可见的进展。
