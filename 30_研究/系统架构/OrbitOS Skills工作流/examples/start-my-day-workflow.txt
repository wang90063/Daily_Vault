start-my-day 工作流示意图
================================

触发: /start-my-day

步骤1: 收集上下文（静默）
┌─────────────────────────────────────────┐
│ 1. 获取今天日期: 2026-03-05            │
│ 2. 读取昨天日记: 10_日记/2026-03-04.md │
│    - 提取未完成任务: [ ] 任务1, [ ] 任务2│
│ 3. 搜索活跃项目: 20_项目/**/*.md       │
│    - status: active                     │
│    - 提取当前阶段、待办任务、最后更新   │
│ 4. 检查收件箱: 00_收件箱/*.md          │
│    - status: pending                    │
│    - 统计待处理数量                     │
│ 5. 分析优先级:                          │
│    - 时间敏感项（截止日期）             │
│    - 停滞项目（3天未更新）              │
│    - 逻辑下一步                         │
└─────────────────────────────────────────┘
           ↓

步骤2: 用户交互
┌─────────────────────────────────────────┐
│ AskUserQuestion 工具:                   │
│ Q1: "今天的主要目标是什么？"            │
│     选项: [[项目A]], [[项目B]], 其他    │
│ Q2: "有什么新想法或任务吗？"            │
│     自由文本输入                        │
│ Q3: "有什么阻碍或顾虑吗？"              │
│     自由文本输入                        │
└─────────────────────────────────────────┘
           ↓

步骤3: 创建/更新今日笔记
┌─────────────────────────────────────────┐
│ 检查: 10_日记/2026-03-05.md 是否存在？ │
│ ├─ 存在: Read → 保留内容 → Edit        │
│ └─ 不存在: Read 模板 → Write            │
│                                         │
│ 填充内容:                               │
│ - 待办事项: 昨天未完成 + 用户焦点 +    │
│             项目下一步                  │
│ - 日志: [留空]                          │
│ - 备注: 时间敏感项、停滞项目、收件箱数  │
│ - 相关项目: [[项目A]] - 状态            │
└─────────────────────────────────────────┘
           ↓

步骤4: 处理新想法（来自Q2）
┌─────────────────────────────────────────┐
│ 对于每个新想法:                         │
│ 1. 检查是否已存在于项目/收件箱          │
│ 2. 如为新想法:                          │
│    Write: 00_收件箱/[简短标题].md       │
│    Frontmatter:                         │
│      created: 2026-03-05                │
│      status: pending                    │
│      source: start-my-day               │
└─────────────────────────────────────────┘
           ↓

步骤5: 呈现总结
┌─────────────────────────────────────────┐
│ ## 早安! 今日规划已就绪                 │
│                                         │
│ **今日笔记:** [[2026-03-05]]           │
│                                         │
│ **待办事项:**                           │
│ - [ ] 完成项目A的设计文档               │
│ - [ ] 研究技术B                         │
│ - [ ] 回顾昨天的代码                    │
│                                         │
│ **正在进行项目 (2):**                   │
│ - [[项目A]] - 设计阶段                  │
│ - [[项目B]] - 开发阶段                  │
│                                         │
│ **已记录新想法 (1):**                   │
│ - [[探索技术C的可行性]]                 │
│                                         │
│ **收件箱:** 3 条待处理                  │
│                                         │
│ 准备开始! 快捷操作:                     │
│ - `/kickoff` - 将收件箱条目转为项目     │
│ - `/research` - 深入研究某个主题        │
└─────────────────────────────────────────┘

文件操作总结:
- Read: 10_日记/2026-03-04.md, 20_项目/**/*.md, 00_收件箱/*.md, 99_系统/模板/Daily_Note.md
- Write/Edit: 10_日记/2026-03-05.md, 00_收件箱/[NewIdea].md
- Bash: grep, find (搜索活跃项目和待处理收件箱)

关键特点:
✓ 单代理直接执行
✓ 最小化用户交互（仅3个问题）
✓ 快速反馈（无等待）
✓ 智能优先级排序
✓ 延续未完成任务
