# 技能工作流模式

## 定义

技能工作流模式是指在 OrbitOS 等智能代理系统中，设计和实现自动化工作流的可复用架构模式。这些模式定义了如何组织代理行为、管理用户交互、操作文件系统，以及协调多个代理之间的协作。

## 要点

- **模式分类**: 根据复杂度和交互方式，工作流可分为单代理直接执行、双代理规划-执行、交互式对话、技能组合等模式
- **文件驱动**: 使用标准化的文件结构和 frontmatter 元数据实现状态管理和工作流协调
- **工具链整合**: 灵活组合 Read、Write、Edit、Bash、WebSearch、Task 等工具完成任务
- **用户体验优先**: 在关键决策点提供检查点，保持快速反馈，提供可操作的建议
- **可组合性**: 技能可以相互调用，形成更复杂的工作流链

## 示例

### 模式1: 单代理直接执行

**适用场景**: 简单、快速的工作流，不需要用户中间审阅

**代表技能**: `start-my-day`

**流程**:
```
用户命令 → 收集上下文 → 用户输入 → 创建/更新文件 → 报告结果
```

**特点**:
- 一次性执行，无中间等待
- 最小化用户交互（仅必要问题）
- 直接文件操作
- 快速反馈

### 模式2: 双代理规划-执行

**适用场景**: 复杂工作流，需要深思熟虑的规划和用户审阅

**代表技能**: `kickoff`, `research`

**流程**:
```
用户命令 → Planning Agent → 计划文件 → 用户审阅 → Execution Agent → 报告结果
```

**特点**:
- 两阶段执行，用户检查点
- 全新执行上下文，Token 效率高
- 更好的输出质量
- 详见 [[双代理模式]]

### 模式3: 交互式对话

**适用场景**: 探索性工作，需要多轮对话和思维发散

**代表技能**: `brainstorm`

**流程**:
```
用户命令 → 对话循环 → 综合总结 → 用户选择行动 → 执行选定工作流
```

**特点**:
- 多轮交互，深入探索
- 心理追踪洞察，延迟文件创建
- 灵活的后续行动选项
- 可无缝调用其他技能

### 模式4: 技能组合

**适用场景**: 需要多个技能协同完成复杂任务

**代表示例**: `brainstorm` → `kickoff`

**流程**:
```
Skill A → 输出/总结 → Skill B (通过 Task 工具) → 最终结果
```

**特点**:
- 技能链式调用
- 上下文传递（如头脑风暴总结传递给 kickoff）
- 模块化设计，每个技能保持独立
- 通过标准接口协作

## 设计原则

1. **单一职责**: 每个技能专注一个明确的任务
2. **用户控制**: 在关键决策点提供检查点
3. **全新上下文**: 复杂工作流使用子代理保持上下文清洁
4. **快速反馈**: 最小化等待时间，提供进度更新
5. **错误处理**: 优雅处理边缘情况
6. **文件驱动**: 通过 frontmatter 管理状态
7. **可组合性**: 技能可以相互调用和组合
