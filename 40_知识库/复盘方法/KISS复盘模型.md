# KISS复盘模型

KISS 由 Keep / Improve / Stop / Start 组成，重点在于行动层面的日常复盘。它鼓励先确认做得对（Keep），然后在方向虽对的情况下修正细节（Improve），再识别需要停止的低价值行动（Stop），最后为下一轮尝试设定 Clear Start 的实验。Keep 是最关键的一项，确保“从成功迈向成功”，其次才是调整与启动。

这个模型不专注深度根因，而是强调以结果为判据的行动扫描，可与 [[复盘行动闭环]] 配合，将洞察压缩成 1-3 个可验证动作，并在每次开始新的尝试前先做优先级梳理，避免任务线过多。Stop 与 Start 互为制约，防止在停不下来的情况下再开新任务。
